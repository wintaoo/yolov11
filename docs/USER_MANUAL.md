# 使用说明书

## 1. 系统访问

### 1.1 启动系统

**Windows 用户**：双击项目根目录下的 `start.bat`，系统会自动启动后端和前端服务。

启动完成后浏览器会自动打开 http://localhost:8080

**手动启动**：
```bash
# 终端1 - 启动后端
cd backend
python run.py

# 终端2 - 启动前端
cd frontend
npm run dev
```

**停止系统**：双击 `stop.bat` 或关闭两个终端窗口。

### 1.2 界面布局

系统提供两个核心功能标签页：

- **图片检测** — 上传施工平面布置图进行 YOLO 目标检测和规范检查
- **文档分析** — 上传 Word 施工方案文档，提取图片并用 AI 分析

顶部导航栏显示：
- 系统名称和品牌标识
- 当前加载的 YOLO 模型文件名
- 模型加载状态 (加载中/就绪)

## 2. 图片检测

### 2.1 导入图片

1. 点击顶部 "选择文件夹" 按钮
2. 在弹出对话框中选择包含施工图纸的文件夹
3. 系统会自动过滤并导入支持的图片格式 (`.jpg` / `.jpeg` / `.png` / `.bmp`)

### 2.2 查看检测结果

> 切换图片时系统会自动执行检测，无需手动点击"开始检测"。

**三种视图模式**：

| 模式 | 说明 |
|------|------|
| 检测结果 | 显示 YOLO 检测框和类别标签 |
| 原图 | 显示原始图片 |
| 对比视图 | 左右对比原图与检测结果，中间有拖拽滑块 |

**胶片导航**：左侧缩略图列表，可快速切换图片。已检测的图片会有圆点标记。

### 2.3 规则检查

右侧面板底部显示施工规范检查结果：

- 通过项 (绿色 ✓)：符合规范
- 违规项 (红色 ✗)：不符合规范

每条规则包含：
- 规范编号 (如 `1.5.4-1`)
- 所属类别 (如 "加工场"、"塔吊")
- 严重程度 (严重 / 重要 / 一般)
- 违规原因和建议

## 3. 文档分析

### 3.1 上传文档

1. 切换到 "文档分析" 标签页
2. 点击 "选择 .docx 文件" 按钮
3. 选择施工方案 Word 文档
4. 系统自动提取文档中的所有图片

### 3.2 分析图片

**单张分析**：
1. 在左侧图片网格中点击一张图片
2. 右侧详情面板出现
3. 点击 "AI 分析此图" 按钮
4. 等待分析完成，查看结果

**批量分析**：
1. 点击顶部的 "批量分析全部" 按钮
2. 蓝色进度条显示分析进度
3. 逐张完成分析，底部"图片分类统计"实时更新
4. 全部完成后，出现"智能汇总报告"

### 3.3 查看分析结果

每张图片的分析结果包含：

- **AI 摘要** (200-400字)：图片核心内容描述
- **AI 评估**：从完整性、标注清晰度、布局合理性、规范符合性四个维度评价，分 优/良/中/差 四等
- **施工计划** (如有)：起止时间、工序安排
- **尺寸规格** (如有)：构件名称、尺寸、型号、数量

### 3.4 图片分类统计

底部固定显示**整个文档**所有图片的分类统计，包括：

- 13 种图纸类型 (总平面布置图、基础结构图、进度计划图 等)
- "其他" (无法明确分类的图片)

> 这个统计展示的是整体文档情况，不会因选择不同的图片而变化。

### 3.5 智能汇总报告

批量分析完成后，系统调用 Qwen2.5-7B-Instruct 模型生成结构化报告，包含：

1. 总体概述 (项目包含的图纸类型和数量)
2. 各类型图纸统计
3. 关键发现和亮点
4. 存在的问题和风险点
5. 总体评价和建议

## 5. 常见问题

### Q: 模型加载失败

- 检查 `backend/models/best.pt` 是否存在
- 查看终端日志中的错误信息
- 确认 PyTorch 和 ultralytics 版本兼容

### Q: 文档分析一直显示"分析中"

- 检查 `.env` 文件中的 `SILICONFLOW_API_KEY` 是否正确
- 确认 SiliconFlow 账户余额充足
- 查看后端日志: `backend/logs/app.log`

### Q: 大图检测很慢

- 系统已自动将长边限制在 1920px
- 建议将超大 PNG (100MB+) 预先压缩为 JPG 格式

### Q: 规则检查结果不准确

- YOLO 检测可能存在漏检，建议检查原始检测结果
- 规则容差经过调优，但特殊情况可能需要调整阈值
