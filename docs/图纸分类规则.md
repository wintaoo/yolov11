# 图纸分类规则

## 概述

分类系统采用**多信号加权融合**策略，纯字符串关键词匹配，不依赖 AI 语义理解。代码入口：`backend/app/services/classifier_service.py`。

| 1 | 总平面布置图 | 施工总平面、场地布置、总平图 |
| 2 | 基础结构图 | 桩基、筏板、承台、基础平面 |

## 四个信号源

| 信号源 | 权重 | 说明 |
|--------|------|------|
| `figure_name` | 5.0x | 从上下文提取的图名（如"图1 总平面布置图"），最可靠 |
| `stage_pattern` | 5.0x | 检测 `X阶段施工平面布置图` 模式，映射到对应阶段类别 |
| `context` | 1.0x | 图片前后段落文字中的关键词 |
| `filename` | 0.8x | 文件名（信号弱，兜底） |

### figure_name 提取

从图片上下文文字中用正则提取：

```
图1 总平面布置图
图1-1 基础结构图
Figure 1 施工总平面
附图1 临时用电布置图
```

### stage_pattern 阶段映射

匹配 `X阶段施工平面布置图` 模式，将阶段名映射到类别：

| 阶段名包含 | 映射类别 |
|-----------|---------|
| 园林绿化 / 绿化施工 / 绿化种植 / 景观 | 装饰装修图 |
| 土方 / 基坑 | 土方工程图 |
| 基础 | 基础结构图 |
| 主体 | 主体结构图 |
| 装饰装修 / 精装修 | 装饰装修图 |
| 施工准备 | 施工计划图 |
| 临建 / 临时 | 临建设施平面布置图 |
| 其他 | 总平面布置图（默认） |

### 上下文提取 (context)

见 `backend/app/services/docx_service.py` 中的 `_extract_context_before()` 和 `_extract_context_after()`：

- 上文/下文各最多提取 **100 字**
- 若提取途中遇到**另一张图片**，截断到 **80 字**
- 不足时有多少提取多少

### 文件名 (filename)

仅作为兜底信号。直接从图片文件名（如 `image_001.png`）中匹配关键词，权重最低（0.8x）。

## 评分与置信度

### 关键词匹配打分

每类关键词命中即得分，`raw_score = Σ(命中关键词的长度)`：

```python
# 示例："总平面布置图" 和 "场地布置" 命中 "总平面布置图" 类别
# 总平面布置图类得分 = len("总平面布置") + len("场地布置") = 10 + 4 = 14
```

### 加权融合

```
加权得分 = raw_score × 信号权重

总平面布置图 = 14 × 5.0(figure_name) = 70.0
```

### 置信度计算

```
confidence = 最高分类别得分 / 所有类别总得分
```

- 仅单一类别匹配 → confidence 接近 100%
- 多类别竞争 → 分数分散，confidence 降低
- 无任何关键词命中 → 归为「其他」，confidence = 0

## 完整处理流程

```
.docx 文档
    │
    ├→ extract_images_from_docx()  提取图片 + 上下文文字
    │       │
    │       ├→ _extract_context_before()  上文（≤100字）
    │       ├→ _extract_context_after()   下文（≤100字）
    │       └→ extract_figure_name()      提取图名
    │
    ├→ classify()  多信号融合分类
    │       │
    │       ├→ _score_signal()         关键词匹配（figure_name / context / filename）
    │       ├→ _extract_stage_signal()  阶段模式检测
    │       └→ 加权汇总 → 最高分类别 + 置信度
    │
    └→ 分类结果写入 task 数据
```

## 关键函数

| 函数 | 文件 | 作用 |
|------|------|------|
| `classify()` | `classifier_service.py` | 多信号融合分类入口 |
| `_score_signal()` | `classifier_service.py` | 对单段文本做关键词匹配打分 |
| `_extract_stage_signal()` | `classifier_service.py` | 阶段模式检测 |
| `_map_stage_to_category()` | `classifier_service.py` | 阶段名 → 类别映射 |
| `extract_figure_name()` | `docx_service.py` | 从上下文提取图名 |
| `guess_category_with_confidence()` | `docx_service.py` | 便捷封装（调用 classify） |

## 已知限制

1. **纯字符串匹配**：不涉及语义理解，无法识别同义词或隐含类别信息
2. **关键词覆盖率**：若文档使用了不在关键词表中的表述，分类会失败
3. **多图同段**：同一段落多张图片时，上下文可能重复或混淆
4. **页码不准**：python-docx 无法精确提取 Word 页码，前端改为展示图片序号
