# 图纸分类规则

## 概述

分类系统采用**多信号加权融合 + 分层关键词 + 冲突惩罚**策略。纯字符串匹配，不依赖 AI 语义理解。

代码入口：`backend/app/services/classifier_service.py`

## 四个信号源

| 信号源 | 权重 | 说明 |
|--------|------|------|
| `figure_name` | 5.0x | 从上下文提取的图名（如"图1 总平面布置图"），最可靠 |
| `stage_pattern` | 5.0x | 检测 `X阶段施工平面布置图` 模式，映射到对应类别 |
| `context` | 1.0x | 图片前后段落文字中的关键词 |
| `filename` | 0.8x | 文件名（信号弱，兜底） |

### figure_name 提取

从上下文文字中正则提取图名，结果截断到 **20 字符**：

```
图1 总平面布置图
图1-1 基础结构图
Figure 1 施工总平面
附图1 临时用电布置图
```

### stage_pattern 阶段映射

匹配 `X阶段施工平面布置图` / `X阶段平面布置图` 模式：

| 阶段名包含 | 映射类别 |
|-----------|---------|
| 园林绿化 / 绿化施工 / 绿化种植 / 景观 | 装饰装修图 |
| 土方 / 基坑 | 土方工程图 |
| 基础 | 基础结构图 |
| 主体 | 主体结构图 |
| 装饰装修 / 精装修 | 装饰装修图 |
| 施工准备 | 施工计划图 |
| 临建 / 临时 | 临建设施平面布置图 |
| 其他 | 总平面布置图（默认） |

### 上下文提取 (context)

见 `backend/app/services/docx_service.py`：

- 上文/下文各最多提取 **100 字**
- 若提取途中遇到**另一张图片**，截断到 **80 字**
- 不足时有多少提取多少

## 分层关键词计分

每个关键词分为两层，杜绝"长词天然分高"的偏差：

| 层级 | 基础分 | 说明 |
|------|--------|------|
| **anchor** (锚定词) | 3 分 | 高区分度，几乎只在一类出现 |
| **supporting** (辅助词) | 1 分 | 有区分力但可能跨类出现 |

### 独有性加成

预计算每个关键词出现在几个类别。仅出现在**单一类别**的关键词，分数 **×2**：

```
例如：
  "总平面布置" → 仅出现在总平面布置图 → 独有 → 3 × 2 = 6 分
  "平面布置图" → 也出现在总平面布置图 → 非独有 → 1 分
  "配电"       → 仅出现在临时用电布置图 → 独有 → 1 × 2 = 2 分
  "框架"       → 仅出现在主体结构图 → 独有 → 1 × 2 = 2 分
```

### stage_pattern 计分

阶段模式匹配等价于独有锚定词，固定 **6 分**（`3 × 2`），不再按字符串长度计分。

## 置信度计算

```
加权得分 = raw_score × 信号权重
置信度   = 最高分类别得分 / 所有类别总得分
```

### 冲突惩罚

当 **图名（figure_name）与上下文（context）指向不同类别** 时，置信度额外 **×0.5**：

```
例如：
  figure_name → 临时用电布置图 (top-1)
  context     → 临时用水布置图 (top-1)
  → 冲突！ 置信度 ×0.5

  figure_name → 总平面布置图 (top-1)
  context     → 总平面布置图 (top-1)
  → 一致，不惩罚
```

这是为了捕捉图片标题与正文内容矛盾的情况（如标题编号错位、文档结构混乱等）。

## 完整处理流程

```
.docx 文档
    │
    ├→ extract_images_from_docx()  提取图片 + 上下文
    │       │
    │       ├→ _extract_context_before()   上文（≤100字，遇图→80字）
    │       ├→ _extract_context_after()    下文（≤100字，遇图→80字）
    │       └→ extract_figure_name()       提取图名（截断20字）
    │
    ├→ classify()  多信号融合分类
    │       │
    │       ├→ _score_signal()          分层关键词匹配（anchor=3/supporting=1）
    │       ├→ _extract_stage_signal()   阶段模式检测（6分）
    │       ├→ 加权汇总 → 最高分类别
    │       ├→ 冲突惩罚 → figure_name vs context 不一致时 ×0.5
    │       └→ 置信度 = 最高分 / 总分
    │
    └→ 分类结果写入 task 数据
```

## 已知限制

1. **纯字符串匹配**：不涉及语义理解，无法识别同义词或隐含类别信息
2. **关键词覆盖率**：若文档使用了不在关键词表中的表述，分类会失败
3. **多图同段**：同一段落多张图片时，上下文可能重复或混淆
4. **页码不准**：python-docx 无法精确提取 Word 页码，前端改为展示图片序号
